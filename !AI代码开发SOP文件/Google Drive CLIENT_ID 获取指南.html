如何获取 Google Drive API 客户端 ID

一个将静态文档转化为交互式体验的分步指南

  • 步骤 1

    访问 Google Cloud Console

  • 步骤 2

    选择或创建项目

  • 步骤 3

    启用 Google Drive API

  • 步骤 4

    配置 OAuth 同意屏幕

  • 步骤 5

    创建 OAuth 客户端 ID

  • 步骤 6

    获取 CLIENT_ID

步骤 1: 访问 Google Cloud Console

首先，您需要登录到 Google Cloud 的管理界面。

  • 打开您的网络浏览器，访问 Google Cloud Console。
  • 使用您的 Google 账户登录。

步骤 2: 选择或创建项目

API 需要在一个项目中进行管理。您可以选择一个现有项目，或创建一个新项目。

  • 在页面顶部的导航栏中，点击项目选择器（通常显示当前项目名称或“选择项目”）。
  • 选择一个现有项目，或者点击“新建项目”创建一个新项目。
  • 为新项目命名（例如：“我的云端硬盘应用”），然后点击“创建”。

步骤 3: 启用 Google Drive API

为了让您的项目能够使用 Drive 的功能，您必须先启用相关的 API。

  • 项目创建完成后，在左侧导航菜单中，点击“API 和服务” > “库”。
  • 在搜索框中输入 Google Drive API，然后从搜索结果中选择它。
  • 点击“启用”按钮来启用 Google Drive API。

步骤 4: 配置 OAuth 同意屏幕

这是用户授权您的应用访问其数据时看到的界面。

  • 在左侧导航菜单中，点击“API 和服务” > “OAuth 同意屏幕”。
  • 用户类型：
    • 如果您是为自己或组织内部使用，选择“内部”。
    • 如果您是为外部用户或公开应用，选择“外部”。（外部应用需要 Google 审核）
  • 点击“创建”，并填写应用信息（应用名称、用户支持电子邮件等）。
  • 在“范围”步骤中，点击“添加或移除范围”，搜索并添加 .../auth/drive 范围。
  • 如果需要，添加测试用户，然后保存配置。

步骤 5: 创建 OAuth 客户端 ID

客户端 ID 用于向 Google 标识您的应用。

  • 在左侧导航菜单中，点击“API 和服务” > “凭据”。
  • 点击页面顶部的“创建凭据”按钮，然后选择“OAuth 客户端 ID”。
  • 应用类型： 选择“Web 应用程序”。
  • 授权的 JavaScript 源： 这是您的应用程序运行的域。非常重要！
    • 本地开发: http://localhost:3000
    • 已部署: https://your-app-domain.com
  • 授权的重定向 URI： Google 完成授权后将用户重定向到的地址。通常与上面的源相同。
  • 点击“创建”。

步骤 6: 获取 CLIENT_ID

创建成功后，您将看到您的凭据。

  • 创建完成后，屏幕上会弹出一个窗口，显示您的“客户端 ID”和“客户端密钥”（如果适用）。
  • 复制您的“客户端 ID”。它通常以 .apps.googleusercontent.com 结尾。

恭喜！

现在，您已经成功获取了 CLIENT_ID。您可以将其粘贴到您的应用程序中开始使用了。
