工程物料知识库：一套完整的RAG运作流程

本方案旨在为工程行业设计一套完整、高效且可靠的物料知识库系统，基于RAG架构，提供精准、可溯源的查询结果。

一、 核心：数据结构与文件存储策略

1. 数据库结构设计 (Data Structure Design)

成功的关键在于向量数据库中存储的不仅仅是向量，更是包含了丰富上下文的结构化数据。

字段名 (Field) 类型 (Type) 描述 (Description)
chunk_id String 文本块的唯一标识符
vector Array[Float] 文本块内容的语义向量
content String 文本块的完整原始内容（缓存）
metadata JSON Object 极其关键的元数据

3. 源文件存储与交互策略

核心概念：大语言模型（LLM）在回答问题时，从不直接与原始文件交互。

  • 源文件存储：所有原始文件都必须存储在一个稳定、可靠、有权限控制的中央存储位置（如 AWS S3, Google Cloud Storage）。
  • 源文件的作用：仅在“知识库创建时”用于解析，以及在“用户点击引用时”供人类验证。
  • LLM的交互对象：LLM只与数据库中缓存的`content`字段交互，保证查询的高效和稳定。

查询举例

文本查询 (`我要买m6螺丝...`): 系统将执行上述完整流程，最终返回包含供应商和价格的总结性答案，并附上可点击的引用链接。

上传请购单查询: 系统会解析单据中的每一项物料，为每一项都独立执行一次查询流程，最后汇总所有结果，生成一个综合性的建议清单。
